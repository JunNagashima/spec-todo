## 目的（何が嬉しい？）
- タスクカード押下で、APIから詳細を取得し、詳細モーダルが表示されタスク詳細情報を確認できる

## 画面/操作（ユーザーが何をする？）
- タスクカード押下で詳細モーダルが表示
- 詳細モーダルでタスクの詳細情報を確認
- 「x」ボタンまたは、「モーダル枠外」で詳細モーダル非表示

## 受け入れ条件（Given/When/Then で 3〜5 本）
### 正常系

#### 初期表示時
Given 任意のタスクデータを用意する
When 一覧画面を開く
Then タスク詳細モーダルが非表示であること

#### タスク詳細モーダルを表示する
Given 任意のタスクデータを用意する
When タスク一覧画面を開く
And 任意のタスクカードを押下する
Then タスク詳細モーダルが表示する

#### タスク詳細モーダルを非表示する
Given 任意のタスクデータを用意する
And タスク一覧画面を開く
And 任意のタスクカードを押下する
When 「x」ボタンまたは、「モーダル枠外」を押下する
Then タスク詳細モーダルが非表示になること

#### タスク詳細取得し表示する
Given titleが「test1」,dueDateが「2026-06-25」,priorityが「HIGH」,statusが「TODO」,createdAtが「2026-03-12」のタスクデータを用意する
When タスク一覧画面を開く
And 作成したタスクカードを押下する
Then API が呼び出されること
And 処理中は、インジケーターが表示される
And API が 200 を返すこと
And インジケーターが非表示になる
And 詳細モーダルのタイトルが「test1」で表示していること
And 期限が「2026/06/25」で表示していること
And 優先度が「高」で表示していること
And ステータスが「TODO」で表示していること
And 登録日が「2026/03/12」で表示していること

### 異常系

#### データ取得処理で失敗
Given 任意のタスクデータを用意する
And タスク一覧画面を開く
When 任意のタスクカードを押下する
Then 処理中は、インジケーターが表示される
And API が 200 などの成功以外のステータスコードを返す
And インジケーターが非表示になる
And 「取得に失敗しました」のトーストが表示される
And トースト表示 5 秒後に非表示になる
And モーダルを閉じる

## データ（更新されるテーブル/モデル、整合性ルール）
- tasks(参照)
