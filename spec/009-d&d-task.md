## 目的（何が嬉しい？）
- カンバン上でタスクカードをドラッグ&ドロップして status を更新できる（楽観更新）

## 画面/操作（ユーザーが何をする？）
- タスクカードはドラッグできる
- ドロップ可能な場所は「列（TODO/DOING/DONE）」のみ
- 同一列内ドラッグ（列内並び替え）はやらない（順序は変えない）

### 更新の挙動（楽観更新）
- 別列へドロップした瞬間に、カードはドロップ先の列へ即時移動する（UI反映）
- UI反映後に、裏側で status 更新API を実行する
- 更新処理中は当該カードは再ドラッグ不可（多重更新防止）
- 更新に成功した場合、追加の通知は行わない

### 更新失敗時の挙動
- status 更新API が成功以外を返した場合、エラーモーダルを表示する
- エラーモーダル文言：
  - 本文：「更新に失敗しました。最新の状態を取得するため、一覧を再読み込みします。」
  - ボタン：「OK」
- 「OK」押下で一覧画面を再読み込みする（window reload 相当）
- エラーモーダル表示中は背面操作不可

## API
- PATCH /api/tasks/{id}/status
- request: { status: "TODO" | "DOING" | "DONE" }
- response: 200

## 受け入れ条件（Given/When/Then）
### 正常系

#### 別列へドロップすると即時移動し、成功したらそのまま確定する
Given status=TODO のタスクが1件存在する
And 一覧画面を開く
When そのタスクカードを TODO列から DOING列へドラッグ&ドロップする
Then タスクカードは即時に DOING列へ表示される（楽観更新）
And 裏側で status 更新API が実行される
And API が 200 を返す
And カードは DOING列に表示されたままである

#### 同一列へドロップしても何も起きない（今回の仕様）
Given status=TODO のタスクが1件存在する
And 一覧画面を開く
When そのタスクカードを TODO列内へドラッグ&ドロップする
Then APIは実行されない
And タスクカードは TODO列に表示されたままである

### 異常系

#### 更新に失敗した場合、エラーモーダル表示→OKで再読み込みする
Given status=TODO のタスクが1件存在する
And 一覧画面を開く
When そのタスクカードを TODO列から DONE列へドラッグ&ドロップする
Then タスクカードは即時に DONE列へ表示される（楽観更新）
And 裏側で status 更新API が実行される
And API が 200 などの成功以外のステータスコードを返す
Then エラーモーダルが表示される
And エラーモーダル表示中は背面操作不可である
When エラーモーダルの「OK」を押下する
Then 一覧画面が再読み込みされる

## データ（更新されるテーブル/モデル、整合性ルール）
- tasks（更新）
  - update columns: status, updatedAt
